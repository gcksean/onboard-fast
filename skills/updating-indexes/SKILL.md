# Updating Indexes

## Overview
Keep each bucket `INDEX.md` concise, linked, and current after entry changes.

## When to Use
- Any add/edit/delete under `resources/entries`, `concepts/entries`, `procedures/entries`, or `questions/entries`.

## Quick Reference
1. Identify changed entries by bucket.
2. Update only affected `INDEX.md`.
3. Ensure link format: `- [Title](entries/<file>.md) — summary`.
4. Remove duplicate or stale lines.
5. Keep ordering predictable (newest first unless manually curated).

## Common Mistakes
- Updating entry files but forgetting indexes.
- Broken links to moved/renamed files.
- Long index paragraphs that hide scannability.
